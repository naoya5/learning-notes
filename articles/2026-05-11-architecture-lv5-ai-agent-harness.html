    1. AIエージェントの定義と3形態

    「AIエージェント」とは何か。まず定式化する。

    エージェント = LLM + Tools + Memory + Loop

    3形態の比較

    形態特徴例制御性
    Workflow事前定義の手順、決定的LangGraph / ariadne高（管理しやすい）
    Reactive1ステップごとに判断ReAct中
    Autonomous目標から自分で計画AutoGPT / BabyAGI低（暴走リスク）
    2026年の現実: Autonomous エージェントは 商用ではほぼ使われない。理由はコスト爆発と暴走リスク。実用は Workflow と Reactive のハイブリッド。

    2. 主要エージェントパターン

    ReAct（Reasoning + Acting）— 最も基本

    「思考 → 行動 → 観察」を繰り返す。Yao et al.（Princeton, 2022）。

    User: 東京の今の気温を教えて。摂氏華氏両方で。

Thought: 東京の今の気温を知る必要がある。
Action: search_weather("東京")
Observation: 東京の現在気温は摂氏 22度。

Thought: 華氏に変換する。22 × 9/5 + 32 = 71.6
Action: respond("東京は現在 摂氏22度（華氏71.6度）です")

    async function reactAgent(query: string, tools: Tools, llm: LLM) {
  const history: string[] = [];
  for (let i = 0; i < 10; i++) {
    const prompt = `
${query}

過去の思考:
${history.join("\n")}

次の Thought と Action を出力してください。
完了したら Action: finish(answer) を返す。
`;
    const response = await llm.complete(prompt);
    const { thought, action } = parseReAct(response);
    history.push(`Thought: ${thought}`, `Action: ${action.name}(${action.args})`);

    if (action.name === "finish") return action.args.answer;

    const observation = await tools[action.name](action.args);
    history.push(`Observation: ${observation}`);
  }
  throw new Error("最大ステップ超過");
}

    Plan-and-Execute — 計画してから実行

    まず全体計画を立て、各ステップを順次実行。ariadne の方式。

    async function planAndExecute(goal: string, llm: LLM) {
  // Plan
  const plan = await llm.complete(`Goal: ${goal}\nList steps in order:`);
  const steps = parseSteps(plan);

  // Execute
  const results = [];
  for (const step of steps) {
    const result = await executeStep(step);
    results.push(result);
    if (!result.ok) break;  // 早期失敗
  }
  return results;
}

    Tree of Thoughts（ToT）— 複数経路を探索

    複数の思考経路をツリー状に展開し、最良を選ぶ。複雑推論問題で性能向上。コスト高め。

    3. Harness Engineering — 本ガイドの核心 ⭐

    AIエージェントを「正しく動かすための土台」をエンジニアリングする領域。これが無いAIアプリ = 計器無しの飛行機。業界用語で "Vibe Engineering"（雰囲気エンジニアリング）と呼ばれる。

    軸1: Observability（観測性）

    すべての LLM 呼び出しを 記録・可視化する。LangSmith / Langfuse / W&B Weave 等。

interface LLMTrace {
  traceId: string;
  parentId?: string;
  model: string;
  prompt: string;
  response: string;
  inputTokens: number;
  outputTokens: number;
  latencyMs: number;
  costUsd: number;
  success: boolean;
  error?: string;
  occurredAt: Date;
}

class TracedLLM {
  constructor(private llm: LLM, private store: TraceStore) {}

  async complete(prompt: string, traceId: string): Promise<string> {
    const start = Date.now();
    try {
      const result = await this.llm.complete(prompt);
      await this.store.save({
        traceId,
        model: this.llm.model,
        prompt,
        response: result.text,
        inputTokens: result.usage.input,
        outputTokens: result.usage.output,
        latencyMs: Date.now() - start,
        costUsd: calculateCost(result.usage),
        success: true,
        occurredAt: new Date(),
      });
      return result.text;
    } catch (e) {
      await this.store.save({
        traceId, /* ... */ success: false,
        error: String(e),
      });
      throw e;
    }
  }
}

async function runEval(set: EvalCase[], agent: Agent, judge: LLM) {
  const results = [];
  for (const c of set) {
    const response = await agent.run(c.query);
    const score = await judge.score({
      query: c.query,
      response,
      expected: c.expected_topics,
      rubric: ["正確性", "丁寧さ", "完全性"],
    });
    results.push({ id: c.id, score, response });
  }
  return aggregate(results);  // 平均スコア、合格率等
}

    軸3: Safety（ガードレール）

    // 入力フィルタ + 出力フィルタの多層防御
class GuardrailedAgent {
  constructor(
    private agent: Agent,
    private inputFilters: InputFilter[],
    private outputFilters: OutputFilter[],
  ) {}

  async run(input: string): Promise<string> {
    // 入力チェック
    for (const f of this.inputFilters) {
      const verdict = await f.check(input);
      if (!verdict.safe) return `申し訳ございません: ${verdict.reason}`;
    }

    const response = await this.agent.run(input);

    // 出力チェック
    for (const f of this.outputFilters) {
      const verdict = await f.check(response);
      if (!verdict.safe) return "オペレータにおつなぎします";
    }

    return response;
  }
}

// フィルタ実装例
class PIIFilter implements OutputFilter {
  async check(text: string) {
    const phoneRegex = /\d{2,4}-?\d{2,4}-?\d{4}/;
    const emailRegex = /[\w.+-]+@[\w-]+\.[\w.-]+/;
    const ccRegex = /\d{4}-?\d{4}-?\d{4}-?\d{4}/;
    if (phoneRegex.test(text) || emailRegex.test(text) || ccRegex.test(text)) {
      return { safe: false, reason: "PII検出" };
    }
    return { safe: true };
  }
}

class TopicFilter implements OutputFilter {
  banned = ["医療診断", "投薬助言", "法律相談"];
  async check(text: string) {
    for (const t of this.banned) {
      if (text.includes(t)) return { safe: false, reason: t };
    }
    return { safe: true };
  }
}

    class BudgetGuard {
  constructor(
    private store: CostStore,
    private dailyLimit = 50,
    private monthlyLimit = 1000,
  ) {}

  async checkBeforeCall(estimateUsd: number): Promise<boolean> {
    const today = await this.store.todayCost();
    const month = await this.store.monthCost();

    if (today + estimateUsd > this.dailyLimit) {
      await alert("daily budget exceeded");
      return false;
    }
    if (month + estimateUsd > this.monthlyLimit) {
      await alert("monthly budget exceeded");
      return false;
    }
    return true;
  }
}

    4. RAGアーキテクチャ

    「Retrieval Augmented Generation」 — 自社データを検索して LLM に渡す手法。Hallucination を抑える基本。

    コンポーネント詳細

    要素役割技術選択肢
    Loaderファイル読み込みLangChain Loaders / unstructured.io
    Splitterチャンク分割RecursiveCharacterTextSplitter
    Embedder埋め込み生成OpenAI / multilingual-e5
    Vector DB類似検索Pinecone / Weaviate / sqlite-vec
    Retrievertop-K取得Hybrid（BM25+Vector）
    Reranker再順位付けCohere Rerank / Cross-Encoder
    Generator回答生成GPT-4 / Claude

    品質向上テクニック

    テクニック効果
    Hybrid Searchキーワード+ベクトルで精度UP
    Query Expansionクエリを言い換えて広く検索
    HyDE仮回答を埋め込んで検索
    Re-rankingtop-50 → 再評価でtop-5
    Chunk Overlapチャンク境界の情報損失を防ぐ
    Parent Chunk小チャンクで検索、親文脈を返却

    RAG実装コード

    // 1. Indexing
async function index(docs: Document[]) {
  for (const doc of docs) {
    const chunks = splitter.split(doc.content);  // チャンク分割
    for (const c of chunks) {
      const vector = await embedder.embed(c.text);
      await vectorDb.upsert({
        id: c.id,
        vector,
        metadata: { docId: doc.id, text: c.text, parentId: c.parentId },
      });
    }
  }
}

// 2. Hybrid Search
async function hybridSearch(query: string, k = 50) {
  const [vectorHits, bm25Hits] = await Promise.all([
    vectorDb.search(await embedder.embed(query), k),
    fts.search(query, k),  // FTS5 / Elasticsearch
  ]);
  return rrf(vectorHits, bm25Hits);  // Reciprocal Rank Fusion
}

// 3. Rerank
async function rerank(query: string, candidates: Hit[], topN = 5) {
  const scored = await reranker.rank(query, candidates.map(c => c.text));
  return scored.slice(0, topN);
}

// 4. Generate
async function generate(query: string, context: Hit[]) {
  const prompt = `
以下の文脈を参考に質問に答えてください。
文脈になければ「資料にありません」と答えてください。

文脈:
${context.map(c => `[${c.id}] ${c.text}`).join("\n")}

質問: ${query}

回答（必ず引用[id]を付けてください）:
`;
  return await llm.complete(prompt);
}

// 統合
async function answer(query: string) {
  const candidates = await hybridSearch(query, 50);
  const top = await rerank(query, candidates, 5);
  return generate(query, top);
}

    5. マルチエージェント設計

    4つの構成パターン

    Supervisor-Worker（最も実用）

    class Supervisor {
  constructor(
    private workers: Map<string, Agent>,
    private llm: LLM,
  ) {}

  async run(task: string) {
    const plan = await this.llm.complete(`
タスク: ${task}
利用可能なWorker: ${[...this.workers.keys()].join(", ")}
各Workerに振るサブタスクをJSON配列で返してください。
`);
    const subtasks = JSON.parse(plan);

    const results = await Promise.all(
      subtasks.map(async (s) => {
        const w = this.workers.get(s.worker)!;
        return await w.run(s.task);
      }),
    );

    return await this.llm.complete(`
タスク: ${task}
各Workerの結果:
${results.map((r, i) => `[${subtasks[i].worker}] ${r}`).join("\n")}

統合した最終回答:
`);
  }
}

// 使う側
const sup = new Supervisor(
  new Map([

    ["Coder", coderAgent],
    ["Reviewer", reviewerAgent],
  ]),
  llm,
);

    マルチエージェントの落とし穴

    よくある失敗:
    • 会話爆発: エージェントが互いに無限に話す
    • コスト爆発: トークン消費が指数的に
    • 責任不在: 「誰がやるか」が曖昧
    • デバッグ困難: 失敗追跡が困難
    うまくいくパターン:
    1. 明確な役割分担（Researcher / Coder / Tester）
    2. 終了条件の明示（最大ターン数、合意条件）
    3. observerエージェント（暴走を止める）
    4. ariadneのレビュー哲学: 「self-report NGで一次ソース尊重」

    実プロダクト分析: ariadne の Harness 完成度

    なおちゃん自作の ariadne を Harness 7軸で採点する。

    • ✅ compose & remember — LLMで構造化タスク生成 / 実行ログから workflow 自動生成

    Q3. RAG精度を上げるには

    RAGで正答率が60%しか出ない。どう改善する？

    解答を見る

    段階的に検証:

    まとめ — Lv5の要点（&シリーズ全体）

    1. エージェント = LLM + Tools + Memory + Loop
    2. 3形態: Workflow / Reactive / Autonomous（実用は前2つ）
    3. 主要パターン: ReAct / Reflexion / Plan-and-Execute / Tree of Thoughts
    4. Harness Engineering 7軸: Observability / Evaluation / Safety / Cost / A/B / Regression / Alerting
    5. Harness優先順: Observability → Cost → Safety → Eval
    6. RAG: Hybrid Search + Reranking + 出典強制 で品質UP
    7. マルチエージェント: Supervisor-Worker が最も実用的
    8. 落とし穴注意: 会話爆発・コスト爆発・責任不在
    9. Vibe Engineering を避ける: 計器なしで本番投入しない
    シリーズ全体の総括:

    Lv1〜Lv5を1周したあなたは、「AIに正しく指示出すための設計の地図」を頭の中に持っている状態。

    AIは実装は得意、設計判断は苦手。指示する側の解像度がそのまま出力品質になる。

    これからは、AI料理人を雇って指示出しできる料理長として、自分のプロダクトを 設計してAIに任せる 立場で動いていける。
